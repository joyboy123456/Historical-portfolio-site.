# 内容结构计划 - 电商设计师作品集网站

## 1. 材料清单

**内容文件：**
- 研究文件来自 `docs/research/` 目录
- Anthropic官网设计分析（导航、字体、间距参考）
- 作品集案例研究（22+极简案例）
- UI组件研究（卡片、按钮、网格最佳实践）

**视觉素材：**
- 设计师头像（圆形头像用于Hero区）
- 作品展示图片（电商项目截图、UI设计稿）
- Logo（导航栏左侧）
- 社交媒体图标（页脚）

**数据文件：**
- 无需额外数据文件，内容将直接嵌入

## 2. 网站结构

**类型：** SPA（单页应用）

**理由：** 
- 内容结构简洁（5个主要区域）
- 单一目标：展示作品集并引导联系
- 总内容量预计 <2000 字
- 图片数量 <15 张（头像 + 作品卡片）
- 适合流畅滚动的叙事体验

## 3. 页面区域分解

### 区域 1: 导航栏（Navigation）
**目的：** 提供清晰的页面导航和品牌识别

**内容映射：**

| 区域元素 | 组件模式 | 数据来源 | 内容提取 | 视觉素材 |
|---------|---------|---------|---------|---------|
| Logo | Logo Pattern | 无 | 设计师姓名或品牌标志 | 设计师Logo图片 |
| 导航菜单 | Horizontal Nav Links | 无 | 锚点链接："作品 / 关于 / 联系" | - |

**禁止内容：** 
- ❌ 无样式指令（居中、对齐等）
- ✅ 仅内容映射

---

### 区域 2: Hero 区域（首屏）
**目的：** 建立第一印象，展示设计师身份与专业定位

**内容映射：**

| 区域元素 | 组件模式 | 数据来源 | 内容提取 | 视觉素材 |
|---------|---------|---------|---------|---------|
| 头像 | Circular Avatar | 无 | - | 设计师专业头像（圆形裁剪） |
| 标题 | Hero Title Pattern | 无 | 设计师姓名 | - |
| 副标题 | Hero Subtitle | 无 | 一句话介绍（如"电商体验设计师 / 用户界面专家"） | - |
| CTA按钮 | Primary Button | 无 | "查看作品" 或 "联系我" | - |

---

### 区域 3: 作品展示区（Portfolio Grid）
**目的：** 以网格布局展示精选作品项目

**内容映射：**

| 区域元素 | 组件模式 | 数据来源 | 内容提取 | 视觉素材 |
|---------|---------|---------|---------|---------|
| 区域标题 | Section Headline | 无 | "精选作品" | - |
| 作品卡片（6-9个） | Article Card Grid (3列) | 无 | 每个卡片包含：<br>• 项目名称<br>• 项目简介（1-2句）<br>• 项目标签（如"电商设计""移动端UI"） | 每个项目的主视觉图片（3:2或16:9比例） |

**卡片内容结构示例：**
- 项目1：某电商平台首页改版
- 项目2：移动购物车体验优化
- 项目3：品牌视觉系统设计
- 项目4：产品详情页设计
- 项目5：checkout流程优化
- 项目6：设计系统搭建

---

**目的：** 介绍设计师背景、技能与工作方式

**内容映射：**

| 区域元素 | 组件模式 | 数据来源 | 内容提取 | 视觉素材 |
|---------|---------|---------|---------|---------|
| 区域标题 | Section Headline | 无 | "关于我" | - |
| 个人简介 | Body Text (2-3段落) | 无 | • 专业背景<br>• 设计理念<br>• 工作经验 | - |

## 4. 内容分析

**信息密度：** 低
- 作品集网站，内容精简聚焦
- 预计总文字量 <1500 字
- 图片主导，文字辅助

**内容平衡：**
- 图片：10-15 张（约60%）
- 文字：1200-1500 字（约30%）
- 交互元素：按钮、链接（约10%）
- **内容类型：** 视觉主导 + 简洁文字

**内容密度评估：**
- ✅ 符合 SPA 标准（<2000字、<8图、单一目标）
- ✅ 极简主义理念：内容优先、无冗余
- ✅ Editorial 风格适配：重视排版质量、阅读体验
